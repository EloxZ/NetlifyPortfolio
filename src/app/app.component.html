Trivial Pur7

Terminado
Demo Código Cliente Código Servidor

Trabajo de Fin de Grado. Juego web online que recrea el popular juego Trivial Pursuit con los servicios de Siette.org, página usada habitualmente en la UMA para realizar cuestionarios.

Los usuarios de la aplicación se integran como usuarios externos en Siette.org para permitir el uso de sus servicios, permitiendo crear partidas con sus cuestionarios. Cada categoría se debe asociar a uno de los 6 colores.

Se puede iniciar sesión usando nombre/clave: usuario/usuario o prueba/prueba. Para entrar en una partida deben haber dos usuarios, por lo que se debe usar incógnito u otro navegador para crear dos sesiones.

La página consta de tres módulos:

  • Página PHP alojada en Apache donde los usuarios pueden registrarse y jugar. Los usuarios se almacenan en MongoDB, y una vez se inicia sesión, se pide su autenticación o registro a Siette.org. El token de Siette se vincula en MongoDB para que el servidor de juego pueda verificarlo posteriormente.
  • Módulo JS + PixiJS que renderiza el tablero y permite al jugador interactuar con el servidor de juego mediante suscripciones a eventos de Socket.io. El jugador se puede conectar a una sala introduciendo su código o escaneando su QR en dispositivos móviles.
  • Servidor de juego Node.js que conecta a todos los jugadores bajo una sala Socket.io. Gestiona completamente la lógica del juego, permite reconexiones de jugadores y usa los servicios de Siette para recoger las preguntas y solicitar su evaluación. Además, permite introducir comandos para que el desarrollador pueda ver las conexiones activas, a que usuarios pertenecen, las salas creadas y el estado de cada sala.

MyEntertainList

En desarrollo

Mini red social inspirada en MyAnimeList pero con series y películas. El usuario puede hacer un ranking de los títulos que ha visto o desea ver, además de explorar el catálogo, que muestra información de interés como trailers o sinopsis.

Proyecto creado para aprender el framework Next.js con React. Se puede crear una cuenta usando OAuth de Google o GitHub gracias a NextAuth.js.

Características terminadas:

  • Catálogo de películas y series creado con las entradas de TMDb API (themoviedb.org). Muestra las categorías de mayor interés: popular, mejor valorados, recientes... o los resultados de una búsqueda. Se puede ver la información o trailer del título seleccionado.

Por hacer:

  • Autenticación completa, con verificación de email y vinculación a auth provider.
  • Lista personal de títulos, con posibilidad de compartir con seguidores.
  • Sistema de seguidores y feed con la actividad de cada usuario. (posts de seguidores, alertas de nuevos episodios disponibles de series en su lista, etc.)
  • Críticas a títulos.

Chip-8 Emulator

Terminado
Código

Emulador Chip-8 que permite ejecutar juegos programados en este lenguaje. Este lenguaje se creó en los 70s para permitir facilitar la ejecución de videojuegos en procesadores distintos. Aunque técnicamente es un intérprete, este proyecto se suele usar como introducción a la emulación de hardware, pues se aplican los mismos principios.

Esta desarrollado en C++ para ser lo más eficiente posible. Usando la librería SFML para renderizar los gráficos. Cuenta con los quirks necesarios para que funcionen correctamente los juegos más modernos. Se ha testeado el correcto funcionamiento de las instrucciones con la siguiente suite.

Posibles mejoras:

  • Crear una interfaz de usuario para que el usuario pueda cargar juegos fácilmente, descargandolos de una base de datos a ser posible. Cada juego necesita una velocidad de reloj y una configuración de quirks diferente para funcionar como se pretende, existe una API que ha recogido toda esta información.
  • Agregando nuevas instrucciones se pueden crear evoluciones del chip para aumentar la resolución o soportar más de dos colores.
